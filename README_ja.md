## 📌 プロジェクト紹介

Contents Pickは、多様なOTTサービスに分散しているコンテンツ情報を一箇所で確認し、ユーザーの選択をサポートするために制作された情報要約および推薦ウェブサービスです。

現代人が抱える「選択の疲労感」を軽減するために企画され、TMDB APIを通じて信頼性の高い基本情報を提供し、PythonクローリングによってIMDbの詳細レビューデータを収集・提供します。

データ収集(Python)から保存(MySQL)、APIサーバー(Spring Boot)、クライアントアプリ(Flutter)へと繋がる全体的なデータパイプラインを構築し、実際のサービス運用が可能なレベルの統合体験を目標に開発されました。

<br>

## ✨ 主な機能

- **人気コンテンツのキュレーション:** TMDB APIを活用し、「今日のトレンド」や「OTT別ランキング」など、ユーザーが興味を持つような人気映画のリストを視覚的に提供します。
- **レビューデータベースの構築:** Pythonスクリプトを活用してIMDbの詳細レビューデータを収集・加工し、これをDBにロードしてアプリ上で閲覧できるように実装しました。
- **スマート検索およびフィルタリング:** タイトルの一部を入力するだけで検索が可能であり、ジャンル/最新順/評価順など、様々な基準でコンテンツを分類して探索できます。
- **パーソナライズサービス (My Page):** 「お気に入りリスト」および「視聴履歴」機能を通じて、自分だけのコンテンツリストを管理し、作成したレビューの修正や削除が可能です。
- **ユーザー認証システム:** Eメール認証を通じた会員登録、パスワードの暗号化保存(SHA-256)、ログインセッション管理などを実装しました。

## ⚙️ インストール及び実行方法

```bash
本プロジェクトは、Python(データ収集)、MySQL(DB)、Spring Boot(サーバー)、Flutter(アプリ)で構成されています。
正常に動作させるため、必ず以下の順序（DB -> クローリング -> サーバー -> アプリ）を守ってください。

# 1. リポジトリのクローン (Clone)
git clone [https://github.com/Yun-Gi/ContentsPick-Movie-Recommendation.git](https://github.com/Yun-Gi/ContentsPick-Movie-Recommendation.git)

# 2. データベース設定 (MySQL)
MySQLを実行し、'content_pick' スキーマを作成します。
backend/src/resources/application.properties ファイルで、DB接続情報を自身の環境に合わせて修正します。

# 3. データ収集
crawlingフォルダに移動して依存関係をインストールし、クローラーを実行してDBに初期データをロードします。
pip install -r requirements.txt
python crawler.py

# 4. バックエンドサーバーの実行 (Spring Boot)
backendフォルダをIDEで開き、Application.java ファイルを探して実行(Run)します。
サーバーが8080ポートで正常に実行されているか確認します。

# 5. アプリケーションの実行 (Flutter)
frontendフォルダに移動してパッケージをインストールし、アプリを実行します。

```
